# stdlib/methodologies/ — Standard Methodology Definitions

Full `Methodology<S>` definitions for each canonical P-series entry. Each file exports a methodology with its arms, transitions, safety bounds, and terminal conditions.

| File | ID | Description |
|------|----|-------------|
| `p1-exec.ts` | P1-EXEC | Execution methodology — single-arm execution with gate validation |
| `p2-sd.ts` | P2-SD | Software development — multi-arm (planning, implementation, review, refinement) |
| `p3-gov.ts` | P3-GOV | Governance — council facilitation, decision drafting, approval |
| `p3-dispatch.ts` | P3-DISP | Dispatch — task decomposition and parallel agent coordination |
| `p-gh.ts` | P-GH | GitHub — issue triage, PR review, implementation workflow |
| `arms.ts` | Shared | Cross-methodology arm definitions (reusable arm templates) |
