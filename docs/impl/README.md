# Implementation Plans

Working implementation plans for `pv-method`.

This directory contains volatile planning artifacts — step-by-step implementation plans, task breakdowns, and session notes used during active development. Contents (except this README) are gitignored because they change rapidly and are disposable once the work is done.

## Usage

- Create a plan file before starting a chunk of work (e.g., `001-mvp-core.md`)
- Reference the architecture (`docs/arch/`) for structural decisions and the PRD (`docs/prds/001-mvp.md`) for scope
- Plans are consumed and discarded — do not treat them as durable documentation
- If a plan reveals an architectural decision, record it in `docs/arch/`, not here
