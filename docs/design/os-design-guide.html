§ 1 — Foundation

Design Philosophy

Three principles govern every design decision in the OS. They are not aesthetic preferences — each is derived from a specific research finding. Check every new element against all three.

principle 01 Color carries state
Pre-attentive color is processed in <200ms with zero working memory cost (Healey & Enns 2012). Every status is encoded as color only — never text alone, never a conjunction of two features.

Text identifies which thing. Color identifies what state.
principle 02 Contrast separates
Background color contrast creates visual separation without drawing lines. A darker header on a lighter panel body is a separator. A border on top of that is redundancy — and redundancy is extraneous load (Sweller 1988).

The rule: if contrast creates the separation, the line is not needed.
principle 03 Accent borders carry meaning
Left-side accent borders (2px) carry semantic meaning. Bio = GENESIS-authored. Solar = human decision needed. Nebular = council mode.

A full 4-side border says nothing. It just draws a box. Every border in the OS must answer: what does this border mean?
The derivation chain: Sweller's CLT says eliminate extraneous load. Durlach (2004) says peripheral text changes are invisible during active work. Cowan (2001) says 4 chunks max. These three findings produce the three principles above. When a design decision feels unclear, trace it back to these sources.
§ 2 — Foundation

Separation Language

Zero decorative divider lines. Separation happens through three mechanisms — background layering, spacing, and left-accent borders. Here is each mechanism, what it communicates, and when to use it.

Background Layering

Three depth levels. Each step darker creates a natural hierarchy. Use the darkest surface for headers and labels, the mid level for content surfaces, the void for the canvas behind everything.

Spacing as Separation

✓ Correct

Use gap and padding between items. Agent list: 3px gap between cards, 8px container padding. The air between cards is the separator. Each card is a floating object.

✕ Wrong

Add border-bottom to every list row. A grid of lines tells the eye there are rows — which is extraneous information the content already communicates.

Structural Borders — Near-invisible Only

Structural borders (between panels, between major layout regions) use rgba(255,255,255,.04) — 4% white. This is the brand-guide's section divider opacity. It is felt as an edge, not read as a line.

Pre-attentive Status System

Three agent states. Color is the primary encoding channel. Shape (icon) provides redundancy for accessibility. Text labels for status are always redundant and should be omitted.

StateColorAnimationMeaning — action required
Running var(--bio) Pulse 2.8s ease Agent active — no action needed
Blocked var(--solar) Pulse 1.9s (faster urgency) Needs human decision — surfaces to Panel D Zone 1
Done var(--dim2) Static — no animation Complete — no action needed
Single-feature rule: Pre-attentive detection works only for single-feature differences (Wolfe & Utochkin 2019). Color alone is sufficient and is the highest-bandwidth pre-attentive channel. If you add text AND color AND an icon to encode status, you have created a conjunction — which destroys the pre-attentive advantage and forces serial search.
§ 4 — Surfaces

Panel Architecture

Four panels, four attentional modes. The number matches Cowan's 4-chunk working memory limit. Each panel occupies one chunk. A fifth panel would require one of the four to share a chunk — which means it gets deprioritized and cognitively dropped.

PanelAttentional modeContent typeBody background
A — Agents Synchronic status Pre-attentive color cards only rgba(8,14,24,.72)
B — Whiteboard Spatial thinking GENESIS artifacts, plans, diagrams #050d16 + dot grid
C — GENESIS Decision dialogue Chat — the only text-dense panel rgba(10,24,40,.96)
D — Signals Alert + ambient Zone 1: solar alert cards. Zone 2: activity canvas var(--void)
Panel C is the only text-dense panel. The user is actively reading it — it is the decision surface. All other panels communicate pre-attentively. If you find yourself adding text-dense content to panel A or D, it belongs in panel C. When a blocked agent needs a decision, its message migrates from panel A to panel D Zone 1 as a solar alert card.

§ 7 — Typography

Type System

Three typefaces. Each has a domain. Using the wrong typeface for a content type is a structural error — JetBrains Mono for prose makes prose feel like code. Inter for repo names makes them feel like narrative text.

TypefaceDomainOS usage
Space Grotesk Structure, identity, hierarchy Panel headers, artifact titles, modal titles, section headings, logo
Inter Extended reading, body prose Descriptions in artifacts, documentation, anything more than two sentences
JetBrains Mono Data, code, technical identity All UI metadata: repo names, method IDs, step labels, timestamps, section tags, status labels, all panel headers
Default rule: If it identifies a technical artifact (repo, method, step, timestamp), it uses JetBrains Mono. If it is a title or heading, Space Grotesk. If it is more than two sentences, Inter. When in doubt, default to JetBrains Mono — the OS is a technical environment.

Section Label Pattern

Used to introduce sections and group content without lines. Mono, uppercase, letter-spacing .14–.16em, bio color at 65–75% opacity. Provides visual grouping through typography alone.

section label + heading

Body text follows in Inter. No line, no box — the label + heading + space is the group.

Chat Typography

GENESIS chat bubbles use JetBrains Mono exclusively — the content is technical, not prose. 11px / 1.75 line-height. This is the highest line-height in the OS, intentionally generous for extended reading.

§ 8 — Guidelines

Anti-patterns

These patterns appear regularly and each violates a specific principle. The failure mode is named alongside the rule, so the reasoning is traceable.

✓ Contrast separates

Use a darker header background on a lighter panel body. The contrast creates the visual boundary. No border needed between them.

✕ Line on top of contrast

Add border-bottom to .phead when contrast already separates. The line is redundant — extraneous load by definition (Sweller).

✓ Color carries state

Icon color only for agent status. Bio = running. Solar = blocked. The pre-attentive channel fires in <200ms at zero memory cost.

✕ Text duplicates color

Add "RUNNING" next to a bio-colored icon. The color already communicated the state. The text is a conjunction that adds noise, not signal.

✓ Tags need no borders

Use background fill only for tags. The color category is communicated. The tag reads as a colored chip — complete information, minimal ink.

✕ Border around colored fill

Add a matching border on every tag. A background-colored tag with a border is a box drawn around color. The box adds zero information.

✓ Alerts surface to Zone 1

Blocked agents inject a solar alert card to Panel D Zone 1. The panel is empty when nothing is blocked — emptiness is the "all clear" signal.

✕ Text log in Panel D

A scrolling event list in Panel D. Change blindness (Durlach 2004): peripheral text changes are invisible during active work. The log will not be read.

✓ Left accent = meaning

Use a 2px left border with a semantic color. Bio = GENESIS made this. Solar = you need to act. Every border answers "what does this mean?"

✕ Full border = decoration

Add a 4-side colored border to a card for visual style. A full border says nothing specific. It draws a box. Boxes are cognitive noise unless they carry meaning.

✓ 4 panels maximum

New information types get assigned to one of the four existing panels. Ask: is this synchronic state (A), spatial content (B), dialogue (C), or a signal (D)?

✕ Fifth panel

Add a fifth panel for a new information type. This exceeds Cowan's 4-chunk limit. The fifth panel will be deprioritized and mentally dropped during active work.

The test: When adding any element, ask — "does this element let the user think more freely about abstract ideas, or does it give them one more thing to parse?" If it's the latter, it doesn't belong in the interface. It belongs in GENESIS's memory.
