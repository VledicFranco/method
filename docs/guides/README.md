# Guides

Conceptual guides explaining the method system — what methods and methodologies are, how they work, and how to use them. Start with Guide 1 and read in order.

| # | Guide | Audience | What you'll learn |
|---|-------|----------|-------------------|
| 1 | [What is a Method?](01-what-is-a-method.md) | Everyone | The 5-tuple, steps, objectives, and why formalism matters |
| 2 | [What is a Methodology?](02-what-is-a-methodology.md) | Everyone | How methodologies route to methods, transition functions, P1-EXEC as example |
| 3 | [The Meta-Method Family (P0)](03-meta-methods.md) | Method designers | How methods are designed, compiled, evolved, and composed |
| 4 | [Execution Methods (P1)](04-execution-methods.md) | Agent operators | COUNCIL, ORCH, TMP — when to use each, how routing works |
| 5 | [Software Delivery (P2)](05-software-delivery.md) | Delivery teams | The full PRD-to-audit loop, all 7 methods, worked example |
| 6 | [Project Cards & Instantiation](06-project-cards.md) | Project leads | How to parameterize a methodology for your project |
| 7 | [Compilation Gates (G0-G6)](07-compilation-gates.md) | Method designers | What each gate checks, common failures, how to fix them |

## How to read these guides

- **Using the system?** Start with Guides 1-2 (concepts), then 4 or 5 (whichever methodology you're using), then 6 (project cards).
- **Designing methods?** Read all guides in order, especially 3 and 7.
- **Just curious?** Guide 1 is self-contained and gives you the core idea in 5 minutes.
