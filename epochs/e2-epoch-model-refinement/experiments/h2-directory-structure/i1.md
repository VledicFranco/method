# H2 / I1

**Observation:** E2 is running inside the proposed layout right now. Working notes:

- `hypothesis.md` at the epoch level reads well — theme + list of experiments gives
  a clear map of what the epoch is doing at a glance.
- `experiments/hN-{slug}/` as a directory feels natural. The slug makes it
  human-readable without needing to open the file.
- `spec.md` as the per-experiment entry point (vs. `hypothesis.md`) avoids naming
  confusion: "hypothesis" belongs at the epoch level (the theme), "spec" belongs
  at the experiment level (the falsifiable bet).
- Iteration logs (`i1.md`, `i2.md`) at the experiment level are the right granularity —
  one log per method-iteration session.
- File presence as status works: scanning the `experiments/` directory tells you
  exactly what's open vs. closed without needing a manifest.

One refinement from using it: the E1 `iterations/` directory should be understood
as a transitional structure — E1 had only one thread of work, so `iterations/` was
fine. E2+ use `experiments/hN-{slug}/` instead.

**Verdict:** CONFIRMED. Layout works. No further changes.
