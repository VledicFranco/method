# H4 / I1

CLAUDE.md updated. Changes made:

1. **Three-level vocabulary introduced** — epoch / experiment / iteration defined
   at the top of the section, before the methodology steps.

2. **"Hypothesis" → "theme + experiments"** — epoch opens with a theme (concern area)
   and a list of experiments. The word "hypothesis" now belongs at the experiment level
   (spec.md), not the epoch level.

3. **decision.md framed as retrospective** — "what closed, what was learned, what
   carries forward, what's next" replaces the implicit binary verdict framing.

4. **Stopping condition added** — "close when you can write a coherent retrospective,
   not necessarily when every experiment is closed."

5. **Promotion rule added** — ~3 iterations or sub-questions → own epoch.

6. **Status via file presence table added** — proposed / active / closed, no status fields.

7. **method_reload added** to the tool list and MCP config section.

Cold-start readability check: the three-level vocabulary is defined before any
procedural steps, so a new agent reading top-to-bottom gets the mental model
before the instructions.
